## Этап 2 — Trajectory + automatic episode prototype

Дата: 2026-07-17.

### Проверено

- Мяч: короткий/длинный жест, сглаживание, ограничение длины, прямая/кривая траектория, deterministic result, goal/post/crossbar/miss.
- Ввод: touch-first PointerInput, mouse для локальной разработки, лишние касания, `pointercancel`, `touchcancel`, resize во время жеста, blur/focus cancellation.
- Эпизод: Setup → AwaitInput → Simulating → Success/Fail, пауза во время симуляции, рестарт, 10 последовательных рестартов, отсутствие дублирования обработчиков в unit-сценарии.
- FPS: deterministic outcomes проверены для 30, 60 и нестабильного 47 FPS в unit-тестах.
- Mobile smoke: Playwright проверяет портрет, landscape-заглушку, отсутствие console errors и системного скролла.

### Ограничения окружения

- Реальные Android/iPhone и мобильное приложение Яндекса в контейнере недоступны.
- Визуальная сетка ворот временная и будет заменена оригинальным ассетом на следующих этапах.

## Этап 3 — Vertical slice + data-driven levels

Дата: 2026-07-17.

### Проверено автоматически

- Первый уровень: стартовый экран, начало, обучение, первая точка решения, пас, автопродолжение, вторая точка решения, удар, победа, поражение, повтор, переход дальше, пауза и возврат из паузы.
- Стабильность: 10 последовательных рестартов, несколько побед подряд, несколько поражений подряд, быстрые повторные вызовы retry/next, touchcancel-style cancellation, пауза во время ожидания и во время автополёта.
- Data-driven уровни: загрузка 5 уровней, runtime-валидация схемы, duplicate id, missing player, missing target, invalid camera, invalid reward, impossible config, переход между decision points.
- Mobile smoke: portrait start screen, прохождение vertical slice до победы, landscape overlay, отсутствие системного скролла и console errors.
- FPS/детерминизм: базовая траектория уже покрыта тестами 30/60/47 FPS из этапа 2 и не сломана этапом 3.

- Звуки и эффекты временные; финальный audio/visual pass будет отдельным этапом.
